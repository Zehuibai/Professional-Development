RJDBC Connection JDBC (driverClass = "", classPath = "", identifier.quote = NA) JDBC() function creates a new DBI driver that can be used to start JDBC connections. identifier.quote is the character to use for automatically generated SQL statements or defaults to NA if the back-end doesn’t support quoted identifiers. jcc <- JDBC(driverClass="com.ibm.db2.jcc.DB2Driver", classPath = "/home/jupyterlab/.rlang/db2jcc-db2jcc4.jar");
Connect to the RJDBC database dbConnect(driver, host, user="username", password="pwd",...) dbConnect() Connect to a DBMS going through the appropriate authorization procedure. Some implementations may allow you to have multiple connections open, so you may invoke this function repeatedly assigning its output to different objects. The authorization mechanism depends onthe drivers used. '...' indicated the other parameters passed to the method. conn <- dbConnect(jcc, "jdbc:db2://hostname:32733/bludb", user="qdt91544", password="b1fghNO6d23ZgN38", security="ssl")
dbSendQuery() dbSendQuery(conn, statement, ...) dbSendQuery function just submits and synchronously executes the SQL query to the database engine. It does not extract any records. ... indicates other named parameters passed on to methods. query = "SELECT * FROM SYSIBM.SYSSCHEMATA";
rs = dbSendQuery(conn, query);
fetch() fetch(res, n, ...) fetch() function records from a previously executed query df = fetch(rs, -1);
dbDisconnect() dbDisconnect(conn, ...) dbDisconnect() closes the connection, discards all pending work, and frees resources (e.g., memory, sockets). ... indicates other named parameters passed on to methods. dbDisconnect(conn)
